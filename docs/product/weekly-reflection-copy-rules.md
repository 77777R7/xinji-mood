# Weekly Reflection Copy Rules

Status: accepted for V1

Weekly Reflection should feel like a private pattern note, not a system report.

## Required Shape

The card should stay simple:

1. eyebrow: `Rora's note`
2. headline: one insight, not a category label
3. body: what Rora noticed from traces and action memory
4. evidence chip: compact count of traces and action notes
5. bottom row: one remembered helper or one light next focus

## Role Contract

| weeklyReflectionRole | Weekly Meaning |
| --- | --- |
| `early_cue_practice` | The user practiced noticing the body signal earlier. |
| `thought_sorting` | The user separated facts, guesses, and worries. |
| `loop_labeling` | The user named a repeating pattern. |
| `behavioral_nudge` | The user shrank the next step. |
| `evening_release` | The user parked an unfinished thought. |
| `self_compassion` | The user made self-talk a little less harsh. |

## Copy Rules

- Mention the action only as evidence of a shift.
- Prefer "this week" over exact counts inside the body; keep counts in the chip.
- Do not mention raw schema fields.
- Do not say Rora knows the cause.
- Do not suggest medical, therapeutic, or clinical certainty.
- Do not use generic praise.
- Use "can remember" instead of "will always recommend".

## Good Pattern

```text
Headline: You caught the signal earlier.
Body: Work feedback still appeared in your saved traces, but Body Scan gave Rora a small clue: chest tightness may be the first place this loop shows up.
Bottom row: Worth remembering / Body Scan / Rora can keep this close for the next time the cue appears.
```

## Bad Pattern

```text
Headline: Great job completing your action!
Body: You completed 1 action and had 3 traces. Your stress loop is improving.
Bottom row: Continue optimizing your routine.
```

The bad pattern is too generic, too achievement-coded, and too certain.
